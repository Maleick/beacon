# v1.4.0 Release Notes

_Released: 2026-04-13_

AutoShip shipped all 9 of its own self-improvement issues in a single session using itself. Zero manual PRs. Zero manual merges.

---

## New Features

### 🦀 Rust/unsafe routing (#101)

New `rust_unsafe` task type always routes to Claude (Haiku for simple, Sonnet for medium/complex), bypassing Gemini/Codex which have a 100% STUCK rate on unsafe Rust, DLL injection, and Windows-specific patterns.

**`rust_windows` profile detection**: if `Cargo.toml` exists and `src/` contains `#[cfg(windows)]`, AutoShip automatically overrides all routing to prefer Claude.

Keyword-based promotion: issues containing `unsafe`, `DLL`, `cdylib`, `winapi`, `retour`, or `#[cfg(windows)]` in the body/title are promoted to Claude regardless of task type.

### 📝 test_command auto-detection (#102)

`hooks/init.sh` now scans `CLAUDE.md` then `AGENTS.md` for test command patterns at startup:

- `cargo test`, `pytest`, `npm test`, `make test`, `./gradlew test`, `python3 scripts/dev-preflight.py`

Writes `test_command` and `verify_command` to `.autoship/config.json`. Warns if nothing detected. Idempotent — never overwrites an already-set value.

### 🏗️ Project context injection (#103)

New `hooks/extract-context.sh` extracts Patterns/Conventions/Gotchas from `CLAUDE.md` and `AGENTS.md`. Output capped at 3000 chars and written to `.autoship/project-context.md`. Called by `init.sh` at startup.

A `## Project Context` section is injected into every dispatch prompt template (Gemini, Codex, Haiku, Sonnet), so agents arrive with project conventions already loaded.

### ⚕️ Codex fast-fail + stuck tracking (#104)

- **Fast-fail health check**: `codex --version` runs with a 10s timeout at the top of `dispatch-codex-appserver.sh`. Failure writes STUCK immediately without waiting for `stall_timeout_ms`.
- **Per-tool stuck tracking**: `quota-update.sh stuck <tool>` increments `tool_stuck_count` in quota.json. At ≥3 STUCKs, `exhausted: true` is set and a `TOOL_DEGRADED` event is emitted.

### 🎯 Opus pre-dispatch advisor for risky issues (#105)

Gate condition added to dispatch Step 3C:

```
IF complexity == complex
AND (risk:high label OR body/title contains: unsafe, DLL, hook, injection)
OR crosses 3+ crates/modules
→ spawn Opus advisor (≤200 words)
→ prepend JSON brief as ## Architectural Guidance to Sonnet prompt
```

Budget protection: `advisor_calls_today` tracked in quota.json (cap: 10/day, auto-reset daily via `refresh` subcommand).

## Bug Fixes

### 🔧 Codex app-server auto-fallback (#93)

`dispatch-codex-appserver.sh` now detects initialization failure/timeout, marks `codex-spark` exhausted in quota.json, and exits STUCK. `detect-tools.sh` probes `codex app-server help` before recording quota; marks exhausted on failure.

### 🗂️ Monitor state file initialization (#94)

`init.sh` creates `.autoship/.pr-monitor-seen.json` and `event-queue.json` on first run, preventing `monitor-prs.sh` from failing with `set -euo pipefail` on fresh workspaces.

### 🔑 Compound issue sub-key support (#95)

All hooks now accept compound keys like `issue-757a` or `issue-757-1`. Numeric prefix is extracted for GitHub API and event calls.

### ⚡ 20-agent dynamic cap enforcement (#96)

The 50-agent hard cap and 20-agent soft cap are now consistently documented and enforced across orchestrate/dispatch skills, AUTOSHIP.md, README, and the landing page.
