🚀 With AutoShip

/autoship:start

AutoShip reads your issues, classifies each one, picks the best agent, creates an isolated worktree, dispatches with a focused prompt, verifies the result, opens a PR, waits for CI, merges, closes the issue, and immediately starts the next one.

You come back to merged PRs.

Time: you typed one command.

How It Works

Six steps, fully automated, runs in a loop.

1

Classify

Each open issue is classified into one of 7 task types: research, docs, simple_code, medium_code, mechanical, ci_fix, or complex. Labels are applied to GitHub for visibility.

2

Route

Task type + available quota determines the agent. Codex and Gemini are used first to save Claude tokens. Routing is configured in AUTOSHIP.md and hot-reloads on change.

3

Dispatch

A git worktree is created for isolation. The agent gets a focused prompt with the issue body, acceptance criteria, and test command. Codex runs via JSON-RPC app-server — no tmux needed.

4

Verify

When the agent reports COMPLETE, a Sonnet reviewer checks AUTOSHIP_RESULT.md against the actual diff and acceptance criteria. On FAIL: re-dispatches. After 3 failures: Opus advisor re-slices the issue.

5

PR + CI

Reviewer PASS triggers automatic PR creation. monitor-prs.sh watches CI status every 30 seconds. On green: auto-merge with squash. Branch and worktree are cleaned up.

6

Loop

Issue is closed, result archived to .autoship/results/, token spend recorded in the ledger. AutoShip immediately picks the next issue from the queue.

What You Get

Built for real backlogs, not toy demos.

🤖

Multi-agent dispatch

Codex, Gemini, Copilot, Claude Haiku, and Claude Sonnet — AutoShip picks the right tool for each task type and falls back gracefully on quota exhaustion.

⚡

Parallel execution

Up to 20 issues in flight simultaneously. Bash monitors watch every agent every 5 seconds without blocking the orchestrator.

🔍

Verification pipeline

Every result is reviewed against acceptance criteria before a PR opens. Partial implementations fail automatically and trigger re-dispatch.

🗺️

Live routing config

Edit the YAML front matter in AUTOSHIP.md to change which agent handles which task type. Takes effect on the next dispatch — no restart needed.

💾

Durable state

State in .autoship/state.json and GitHub labels. Kill the session, restart, and AutoShip picks up exactly where it left off.

📊

Token ledger

Per-issue and per-session token spend tracked in .autoship/token-ledger.json. /autoship:status shows per-model cost breakdown.
